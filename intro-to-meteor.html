Что такое Meteor?
Метеор - это платформа для быстрой разработки 
веб приложений используя только один язык
программирования - javascript.
JAVASCRIPT!
Что такое Meteor?
NODE.JS 
+
MONGO DB
+
JAVASCRIPT LIBRARIES
Чистый JavaScript
Всё приложение пишется на Javascript
По умолчанию весь код исполняется и на сервере, и на клиенте - 
следовательно не нужно дублировать код для описания одинаковой логики
и одинаковых структур
При желании можно писать на coffeescript
JAVASCRIPT ВЕЗДЕ!
* Illustration from Brian Donghee Shin presentation "Intro to Meteor.js"
Синхронизация данных
Клиентская часть пишется так, как будто она имеет прямой доступ к БД:
изменения данных сохраняются на клиенте и через веб-сокеты синхронизируются
с сервером, изменения на сервере синхронизируются со всеми клиентами
Данные приходят не через REST, а по протоколу DDP
Полезные ссылки:
DDP Specification - https://github.com/meteor/meteor/blob/devel/packages/ddp/DDP.md 
Intro to DDP - https://meteorhacks.com/introduction-to-ddp.html
How to use DDP-Analyser - https://meteorhacks.com/discover-meteor-ddp-in-realtime.html
Живые обновления страницы
Достигается за счет связывания шаблонов с данными 
При изменении данных шаблон рендерится заново (Blaze)
Метеор использует встроенный язык шаблонов spacebars (handlebars), но позволяет
использовать также и другие языки (jade,..)
Полезные ссылки:
How Blaze Works? - https://meteorhacks.com/how-blaze-works.html

7
принципов
Meteor.js
Принципы взяты из официальной документации Meteor.js - http://docs.meteor.com/#/basic/sevenprinciples
Реактивность
Meteor.js основан на парадигме реактивного программирования
Про реактивное программирование - http://www.reactivemanifesto.org/
Передаются только данные
Сервер не отсылает HTML клиенту 
Клиент сам решает как ему отображать данные
За рендеринг страницы отвечает Blaze 
Один язык разработки
Клиентская и серверная часть пишутся на javascript 
База данных доступна везде
На клиенте и на сервере используется одно API для работы с БД 
На сервере используется MongoDB, на клиенте - MiniMongo 
За синхронизацию данных отвечает Meteor
Компенсация задержки
Пользователь не ждет синхронизации данных с сервером
Сперва данные обновляются в локальной БД, потом изменения 
синхронизируются с сервером
Пользовательский интерфейс обновляется в режиме реального времени

Интеграция инструментов
Meteor объединяет популярные инструменты и библиотеки в одну платформу,
а не заменяет их
Meteor является Open-Source проектом
Простота = производительность
Низкий порог входа в технологию
Простой и понятный API
